# Documentación activa del sistema

La carpeta activa y oficial de documentación arquitectónica y operativa es:

- `docs/reconstruction-phase-1`

## Regla de uso

- la documentación nueva debe escribirse ahí
- la documentación fuera de esa carpeta debe tratarse como histórica, transicional o insumo de recuperación
- las definiciones canónicas solo deben conservarse si están validadas contra el código y el esquema actuales

## Punto de entrada recomendado

Empezar por:

- `docs/reconstruction-phase-1/README.md`
- `docs/reconstruction-phase-1/canonical-definitions.md`
- `docs/reconstruction-phase-1/documentation-governance.md`
- `docs/reconstruction-phase-1/legacy-document-inventory.md`
- `docs/reconstruction-phase-1/system-flows.md`

## Nota sobre documentación vieja

La documentación vieja fue reubicada bajo `docs/reconstruction-phase-1/_historical`. Sigue siendo útil como material histórico, pero ya no debe competir con la carpeta activa como fuente de verdad del presente.
