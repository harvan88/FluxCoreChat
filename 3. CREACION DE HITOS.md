Patrón de Creación de Hitos Inteligente (HCI - Hito Creation Intelligence)
CUANDO RECIBAS UNA NUEVA TAREA O ÁREA:

1. EJECUTA ANÁLISIS DIAGNÓSTICO (Fase 1)
   - Escanea código relevante
   - Evalúa complejidad real (no la aparente)
   - Identifica riesgos específicos

2. GENERA HITO INTELIGENTE (Fase 2)
   - Usa la plantilla estructurada
   - Define puntos de éxito MEDIBLES
   - Documenta consideraciones REALES

3. AUTO-VALIDA (Fase 3)
   - Verifica viabilidad con código existente
   - Ajusta scope si es necesario

4. EJECUTA CON ADAPTABILIDAD (Fase 4)
   - Monitorea discrepancias estimado/real
   - Ajusta dinámicamente
   - Documenta aprendizajes

## Ejemplo: Hito VS-ALIGN

**Patrón aplicado**:
1. CIERRE: Verificación de inconsistencias actuales (vector stores)
2. DEFINICIÓN: Objetivo de paridad con API OpenAI
3. TRANSICIÓN:
   - Refactor: Servicio de eliminación unificado
   - Nuevo: Conversores de política de expiración
   - UI: Componente de colección de archivos
4. VERIFICACIÓN: Pruebas de cascada y políticas
5. DOCUMENTACIÓN: Matriz de conversión de políticas

SOLO CONSULTA SI:
- Encuentras contradicción arquitectónica irresoluble
- El riesgo afecta >2 módulos críticos
- Necesitas redefinir objetivos de negocio
